# OS SIXTH PRACTISE: Secure Archive & FUSE

Продвинутая многопоточная утилита командной строки и драйвер виртуальной файловой системы (FUSE) для создания и прозрачного монтирования зашифрованных архивов в Linux. 

## Главные архитектурные особенности
* **Многопоточность:** Конкурентная упаковка файлов пулом из 3 идентичных потоков-воркеров, которые параллельно читают, шифруют и пишут данные без узких мест.
* **Криптография:** Поточное шифрование RC4 с использованием уникальной криптографической соли для каждого файла внутри образа.
* **Защита памяти (Anti-Dump):** Мастер-ключ жестко изолирован в оперативной памяти с помощью системного вызова `mprotect` (PROT_NONE). Доступ к ключу открывается строго на доли миллисекунды в момент шифрования.
* **Виртуальная ФС (FUSE):** Возможность примонтировать зашифрованный образ как обычную папку и читать файлы с расшифровкой "на лету" без извлечения на диск.

## Установка и сборка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/vanger2607/MIREA_OS/
cd MIREA_OS
```

2. Соберите основную утилиту упаковщика (`secure_copy`):
```bash
make all
```

3. Соберите драйвер FUSE (`fuse`):
```bash
make -f Makefile.fuse all
```

4. Для установки динамической библиотеки криптографии в систему выполните (потребуются права root): 
```bash
make install
```

## Примеры использования

### Работа с CLI-утилитой
Создание зашифрованного архива:
```bash
./secure_copy -add -key "my_password" -image vault.img file1.txt /path/to/folder/
```

Просмотр списка файлов в архиве:
```bash
./secure_copy -list -image vault.img
```

Извлечение конкретного файла:
```bash
./secure_copy -get -key "my_password" -image vault.img -out extracted.txt /file1.txt
```

### Монтирование через FUSE
Создайте пустую директорию и примонтируйте в нее образ:
```bash
mkdir -p /tmp/my_vault
./fuse -key "my_password" -image vault.img /tmp/my_vault -f
```
*Теперь вы можете открывать файлы в `/tmp/my_vault` обычными программами (cat, nano, видеоплееры).*

Для безопасного размонтирования диска используйте:
```bash
fusermount -u /tmp/my_vault
```

## Тестирование и очистка

Запуск полного комплекса тестов (проверка целостности хэшей, краевых случаев и перехвата SIGSEGV):
```bash 
make test_all
```

Для очистки директории от скомпилированных объектов (`.o`, `.so`) и тестовых файлов:
```bash
make clean
make -f Makefile.fuse clean
```
